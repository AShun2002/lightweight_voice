#  测试记录

# 一、边界测试

## 1\.1 正常语音输入

|测试项|输入|预期结果|是否通过|
|---|---|---|---|
|标准普通话语音|清晰普通话录音"你好"|ASR识别为"你好"，回复正常问候，返回音频正常|通过|
|正常语速语音|正常语速日常对话|识别准确，语义理解正确，回答相关|通过|
|标准mp3格式|mp3格式音频文件|正常识别，无格式错误，返回 code=0|通过|

## 1\.2 无音源输入

|测试项|输入|预期结果|是否通过|
|---|---|---|---|
|完全静音文件|全静音mp3文件（3秒）|返回 code=2001 或识别为空，服务不崩溃|通过|
|极短音频|时长不足0\.5秒的音频|友好提示或识别失败，不返回500错误|通过||
|纯噪音无人声|只有背景噪音|识别结果为空或提示未检测到语音|通过|
|空文件|0字节文件|返回参数错误或识别失败，服务不崩溃|通过|

# 二、AI CRM 接口功能测试

测试接口：POST /api/ask/crm

格式：样例问答输入 → 校验正确性

## 测试用例 1：法律问题 → 调用 rag\_law 工具

**样例输入：**

```json
{
  "text": "中国劳动法规定的法定节假日有哪些？"
}
```

**校验正确性：**

-  返回 code = 0，请求成功

-  structured\.tool\_used 包含 "rag\_law"

-  structured\.legal\_reference 不为空，有具体法条内容

-  reply\_text 回答正确，引用了《劳动法》相关内容

-  structured\.confidence 有值（0\-1之间）

## 测试用例 2：数据库查询 → 调用 sql\_agent 工具

**样例输入：**

```json
{
  "text": "查询所有用户的信息"
}
```

**校验正确性：**

-  返回 code = 0，请求成功

-  structured\.tool\_used 包含 "sql\_agent"

-  structured\.sql\_result 不为空

-  返回4条用户记录（张三、李四、王五、赵六）

-  每条记录包含 name、email、age、city 等字段

## 测试用例 3：条件查询 → 调用 sql\_agent 工具

**样例输入：**

```json
{
  "text": "北京有多少个用户？"
}
```

**校验正确性：**

-  structured\.tool\_used 包含 "sql\_agent"

-  正确返回数量：1个（张三，北京）

-  SQL Agent 能正确执行 WHERE 条件查询

## 测试用例 4：商品表查询 → 调用 sql\_agent 工具

**样例输入：**

```json
{
  "text": "最贵的电子产品是什么？价格多少？"
}
```

**校验正确性：**

-  structured\.tool\_used 包含 "sql\_agent"

-  正确识别 products 表

-  正确返回：笔记本电脑，5000元

-  能按 category 过滤和 price 排序

## 测试用例 5：网络搜索 → 调用 tavily\_search 工具

**样例输入：**

```json
{
  "text": "最近有哪些关于劳动法的新规定？"
}
```

**校验正确性：**

-  structured\.tool\_used 包含 "tavily\_search"

-  structured\.search\_result 不为空

-  搜索结果包含标题、链接、摘要

-  若API Key未配置，返回友好错误提示，不崩溃

## 测试用例 6：普通闲聊 → 不调用工具

**样例输入：**

```json
{
  "text": "你好，今天心情怎么样？"
}
```

**校验正确性：**

-  structured\.tool\_used 为空或 null

-  structured\.legal\_reference = null

-  structured\.sql\_result = null

-  structured\.search\_result = null

-  reply\_text 为正常的闲聊回复

## 测试用例 7：多轮对话记忆

**第1轮输入：**

```json
{
  "text": "我叫张三，来自北京",
  "session_id": "test_memory_001"
}
```

**第2轮输入：**

```json
{
  "text": "我来自哪里？",
  "session_id": "test_memory_001"
}
```

**校验正确性：**

-  第2轮能正确回答"北京"

-  对话记忆生效（LangGraph SQLite 记忆）

-  两轮 session\_id 一致

## 测试用例 8：语音输入 CRM 问答

**样例输入：**

上传语音文件（内容为"查询所有用户"），audio=xxx\.mp3

**校验正确性：**

-  ASR 识别正确，user\_text 为"查询所有用户"或近似语义

-  正确调用 sql\_agent 工具

-  返回结构化结果正常

-  audio\_url 不为空（默认返回语音）

# 三、客户管理接口功能测试

测试接口：`/api/customers/*` 系列接口

格式：样例请求 → 校验正确性

## 测试用例 1：新增客户

**样例输入：**

```json
POST /api/customers
{
  "name": "张三",
  "phone": "13800138000",
  "email": "zhangsan@example.com",
  "company": "某某科技有限公司",
  "source": "朋友介绍",
  "remark": "对产品感兴趣"
}
```

**校验正确性：**

- 返回 `code = 0`，请求成功
- 返回的客户信息包含 `id=1`，`name="张三"`
- 自动生成 `created_at`、`updated_at` 时间
- 初始状态为 `"潜在客户"`

## 测试用例 2：查询客户列表

**样例输入：**

```json
GET /api/customers
```

**校验正确性：**

- 返回 `code = 0`，请求成功
- `total` 字段正确（新增后为 1）
- `customers` 数组包含刚新增的张三
- 按创建时间倒序排列

## 测试用例 3：修改客户状态

**样例输入：**

```json
PUT /api/customers/1/status
{
  "status": "意向客户"
}
```

**校验正确性：**

- 返回 `code = 0`，请求成功
- 返回的客户 `status` 字段更新为 `"意向客户"`
- `updated_at` 时间自动更新

## 测试用例 4：添加跟进记录

**样例输入：**

```json
POST /api/customers/1/follow
{
  "content": "今天电话沟通，约了下周一见面",
  "follow_type": "电话",
  "follower": "销售小李"
}
```

**校验正确性：**

- 返回 `code = 0`，请求成功
- 返回跟进记录包含 `id`、`customer_id=1`
- `follow_time` 自动生成当前时间
- 跟进内容、跟进人正确

## 测试用例 5：查询客户详情（含跟进记录）

**样例输入：**

```json
GET /api/customers/1
```

**校验正确性：**

- 返回 `code = 0`，请求成功
- 返回客户张三的完整信息
- `follow_records` 数组不为空，包含刚添加的跟进记录
- 跟进记录按时间倒序排列

## 测试用例 6：按状态筛选客户

**样例输入：**

```json
GET /api/customers?status=意向客户
```

**校验正确性：**

- 返回 `code = 0`，请求成功
- 只返回状态为 `"意向客户"` 的客户
- 其他状态的客户不出现

## 测试用例 7：关键词搜索客户

**样例输入：**

```json
GET /api/customers?keyword=张三
```

**校验正确性：**

- 返回 `code = 0`，请求成功
- 返回姓名 / 电话 / 公司包含 `"张三"` 的客户
- 不相关客户不出现

## 测试用例 8：删除客户

**样例输入：**

```json
DELETE /api/customers/1
```

**校验正确性：**

- 返回 `code = 0`，请求成功
- 再查询客户列表，张三已被删除
- 对应的跟进记录也被级联删除
