# AI CRM 语音交互系统 - 项目说明文档

## 一、项目概述
本项目是在轻量级语音交互服务的基础上，集成 AI CRM Agent，实现语音交互的客户管理系统。用户可以通过语音或文字与系统对话，系统能自动识别用户意图，调用相应的工具（法律检索、数据库查询、网络搜索），并返回结构化的回答。

## 三、功能模块

### 3.1 语音能力
- **ASR 语音识别**：将语音转成文字
- **TTS 语音合成**：将文字回复转成语音
- **支持格式**：mp3、webm 等常见音频格式

### 3.2 CRM Agent 能力
基于 LangGraph 状态图构建的多工具 AI Agent，支持：
- **法律 RAG 检索**：基于《劳动法》等法律文本，检索相关法律条文
- **数据库查询**：用自然语言查询 SQLite 数据库（用户表、商品表）
- **网络搜索**：通过 Tavily 搜索最新资讯
- **对话记忆**：支持多轮对话，记住上下文
- **结构化输出**：返回包含工具使用、法律条文、SQL 结果等的结构化信息

### 3.3 前端交互
- 支持普通模式和 CRM 模式切换
- 语音录制和播放
- 结构化结果展示

### 3.4 客户管理模块
完整的客户生命周期管理功能，支持：
- **客户CRUD**：新增、查询、修改、删除客户
- **客户状态管理**：潜在客户 → 意向客户 → 成交客户 → 流失客户，状态流转
- **跟进记录**：记录每次跟进内容、跟进方式、跟进人，自动关联客户
- **筛选查询**：支持按客户状态、关键词筛选客户列表

## 四、技术栈

| 层级     | 技术                          |
|----------|-------------------------------|
| 后端框架 | FastAPI                       |
| Agent框架 | LangGraph（StateGraph 低层 API） |

## 五、使用说明

### 5.1 启动服务

```bash
# 安装依赖
pip install -r requirements.txt
# 配置 API Key
# 修改 agents/crm_agent/llms.py 中的 API Key
# 启动服务
python api.py
```

### 5.2 访问方式
前端页面：http://localhost:8000
API 文档：http://localhost:8000/docs

### 5.3 模式切换
普通模式：通用语音问答，调用基础 LLM
CRM 模式：AI CRM Agent，支持多工具调用和结构化输出

## 六、核心接口

### 6.1 CRM 问答接口
路径：POST /api/ask/crm
功能：CRM Agent 问答，支持文本和语音输入

参数：
- text：文本内容（和 audio 二选一）
- audio：语音文件（和 text 二选一）
- session_id：会话 ID（可选）
- return_audio：是否返回语音（默认 true）

返回：
- session_id：会话 ID
- user_text：用户输入的文字
- reply_text：AI 回复文字
- audio_url：语音回复地址
- structured：结构化结果（工具使用、法律条文、SQL 结果等）

### 6.2 客户管理接口
| 接口路径 | 方法 | 功能 |
|---------|------|------|
| `/api/customers` | GET | 查询客户列表，支持按状态、关键词筛选 |
| `/api/customers/{customer_id}` | GET | 查询客户详情，包含跟进记录 |
| `/api/customers` | POST | 新增客户 |
| `/api/customers/{customer_id}` | PUT | 修改客户信息 |
| `/api/customers/{customer_id}/status` | PUT | 修改客户状态 |
| `/api/customers/{customer_id}` | DELETE | 删除客户 |
| `/api/customers/{customer_id}/follow` | POST | 添加跟进记录 |
| `/api/customers/{customer_id}/follow` | GET | 查询客户跟进记录 |
